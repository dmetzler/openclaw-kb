# Wiki Management

OpenClaw KB maintains a wiki of Markdown pages organised into four subdirectories. Each page represents a knowledge entity extracted during ingestion and is linked to the SQLite knowledge graph via a `kg_id` in its frontmatter.

## Directory Structure

```
wiki/
├── entities/        # Named things: people, companies, products
├── concepts/        # Abstract ideas: "machine learning", "REST API"
├── topics/          # Broad subjects: "artificial intelligence"
├── comparisons/     # Comparative analyses: "React vs Vue"
├── index.md         # Auto-generated page listing
└── log.md           # Chronological operation log
```

Each subdirectory corresponds to an entity type in the knowledge graph:

| Directory | Entity Type | Description |
|-----------|-------------|-------------|
| `entities/` | `entity` | Named real-world things |
| `concepts/` | `concept` | Abstract ideas and definitions |
| `topics/` | `topic` | Broad subject areas |
| `comparisons/` | `comparison` | Side-by-side analyses |

## Page Format

Every wiki page is a Markdown file with YAML frontmatter:

```markdown
---
id: acme-corp
type: entity
created: '2025-01-15T10:30:00.000Z'
updated: '2025-01-16T14:20:00.000Z'
sources:
  - 2025-01-15-acme-corp-profile.md
confidence: 0.85
related: []
kg_id: 42
---

# Acme Corp

Acme Corp is a technology company specialising in cloud infrastructure...
```

### Frontmatter Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Slugified page identifier (filename without `.md`) |
| `type` | `string` | One of: `entity`, `concept`, `topic`, `comparison` |
| `created` | `string` | ISO 8601 timestamp of page creation |
| `updated` | `string` | ISO 8601 timestamp of last update |
| `sources` | `string[]` | Raw source filenames that contributed to this page |
| `confidence` | `number` | Confidence score (0–1), weighted average across sources |
| `related` | `string[]` | Wikilinks to related pages |
| `kg_id` | `number` | Corresponding entity ID in the SQLite knowledge graph |

## Page Naming

Pages are named using a slugification algorithm:

1. Convert to lowercase
2. Replace spaces with hyphens
3. Strip non-alphanumeric characters (except hyphens)
4. Collapse consecutive hyphens
5. Trim leading/trailing hyphens
6. If longer than 80 characters, truncate to 73 + `-` + 6-char MD5 hash

If a name collision occurs across wiki subdirectories, the entity type is appended as a suffix (e.g. `node-entity.md` vs `node-concept.md`).

### Examples

| Input | Slug |
|-------|------|
| `Acme Corp` | `acme-corp.md` |
| `Machine Learning` | `machine-learning.md` |
| `React vs Vue` | `react-vs-vue.md` |
| `A Very Long Entity Name That Exceeds...` | `a-very-long-entity-name-that-exc...-a1b2c3.md` |

## Creating and Updating Pages

Pages are created automatically during [ingestion](ingestion.md). The pipeline:

1. Extracts entities from source content via LLM
2. Checks if a wiki page already exists for each entity (`findPage`)
3. Creates new pages (`createPage`) or merges updates into existing ones (`updatePage`)

### Creating a Page Programmatically

```js
import { createPage } from './src/wiki.mjs';

const result = createPage(
  {
    name: 'acme corp',
    type: 'entity',
    description: 'A technology company...',
    attributes: { industry: 'tech', founded: 2010 },
  },
  '2025-01-15-source.md'
);

console.log(result);
// { fileName: 'acme-corp.md', filePath: 'wiki/entities/acme-corp.md', kgId: 42 }
```

### Updating an Existing Page

When new information about an entity is ingested, the existing page is updated:

- The `updated` timestamp is refreshed
- The new source file is added to `sources`
- The `confidence` score is recalculated as a weighted average
- New content is appended under an `## Update from <source>` heading
- Duplicate content is not appended (simple dedup)

```js
import { updatePage } from './src/wiki.mjs';

const result = updatePage('acme-corp.md', newEntity, 'new-source.md');
```

## Reading Pages

```js
import { readPage } from './src/wiki.mjs';

const page = readPage('acme-corp.md');
// {
//   data: { id: 'acme-corp', type: 'entity', ... },
//   content: '\n# Acme Corp\n\nA technology company...\n',
//   filePath: 'wiki/entities/acme-corp.md'
// }
```

## Finding Pages

Search across all wiki subdirectories by entity name:

```js
import { findPage } from './src/wiki.mjs';

const found = findPage('Acme Corp');
// { fileName: 'acme-corp.md', filePath: 'wiki/entities/acme-corp.md', type: 'entity' }
```

`findPage` also checks for disambiguated names (with type suffix).

## Wiki Index

The `wiki/index.md` file is auto-generated by `regenerateIndex()`. It lists all pages grouped by type with Obsidian-compatible wikilinks:

```markdown
# Wiki Index

*Auto-generated. Do not edit manually.*

## Entities

- [[acme-corp|Acme Corp]]
- [[alice|Alice]]

## Concepts

- [[machine-learning|Machine Learning]]

## Topics

- [[artificial-intelligence|Artificial Intelligence]]

## Comparisons

(empty)
```

The index is regenerated automatically after each ingestion operation.

## Operation Log

Every ingestion operation is logged in `wiki/log.md`:

```markdown
## 2025-01-15T10:30:00.000Z — URL Ingestion

**Source**: https://example.com/article
**Raw file**: 2025-01-15-article-title.md
**Pages created**: [[acme-corp]], [[alice]]
**Pages updated**: (none)

---
```

The log records:

- Timestamp and source type (URL or text)
- Source URL or manual title
- Raw file name
- Pages created and updated (as wikilinks)
- Any failed pages with error messages

## Obsidian Integration

The `wiki/` directory is designed to work as an [Obsidian](https://obsidian.md) vault. To set it up:

### Quick Setup

1. Open Obsidian
2. Choose **Open folder as vault**
3. Select the `wiki/` directory
4. Done — all pages are immediately browsable

### What Works Out of the Box

- **Wikilinks**: `[[page-name]]` links in `index.md` and `log.md` are standard Obsidian wikilinks
- **Frontmatter**: Obsidian natively renders YAML frontmatter in the Properties panel
- **Search**: Obsidian's built-in search works across all wiki pages
- **Graph view**: Obsidian's graph view shows relationships between pages via wikilinks
- **Folders**: The `entities/`, `concepts/`, `topics/`, `comparisons/` subdirectories appear as folders in the file explorer

### Recommended Obsidian Settings

| Setting | Value | Why |
|---------|-------|-----|
| **Default location for new notes** | `entities/` | Most new pages are entities |
| **New link format** | Shortest path when possible | Keeps wikilinks clean |
| **Use Wikilinks** | On | Matches the link format used by OpenClaw |
| **Detect all file extensions** | Off | Only show `.md` files |

### Editing Pages in Obsidian

You can freely edit wiki pages in Obsidian. Keep in mind:

- **Frontmatter**: Don't remove or rename frontmatter fields — the system relies on `kg_id`, `sources`, and `confidence`
- **Body content**: Edit freely. New ingestion updates are appended, not overwritten
- **File names**: Renaming files in Obsidian will break the link to the knowledge graph. Use `findPage` to locate pages instead of relying on file paths
- **New pages**: Pages created manually in Obsidian won't have a `kg_id` and won't appear in knowledge graph searches until they are ingested
